# AGENTS.md

## Purpose
- This repository is a lightweight GitHub Pages project archive.
- It is not the main submission portfolio and does not replace a resume or PDF portfolio.
- Its job is to let viewers scan selected work quickly through short summaries, visuals, and links.

## Content Principles
- Prefer concise, high-signal summaries over detailed case studies.
- Keep the archive organized year by year.
- Show enough context to understand the work, stack, and outcome without turning each project into a long narrative.
- Use resume or PDF materials for formal submission-oriented detail.

## Editing Guidelines
- Keep the site simple, scannable, and easy to maintain.
- Preserve chronological grouping when adding or revising projects.
- Prefer clear project labels such as company, side project, capstone, or competition when that improves scanability.
- Prioritize visible outcomes, concrete responsibilities, and working links.
- Avoid adding sections or copy that make the site feel like a heavy recruiter-facing portfolio.
- If a detail is technically interesting but too long for this site, leave it for README, resume, PDF portfolio, or interview discussion.

## Writing Style
- Use short descriptions and compact bullets.
- Prefer concrete wording over abstract claims.
- When possible, anchor explanations in what was built, shipped, trained, integrated, or verified.
- If there is a tradeoff between completeness and readability, choose readability.

## Visual Direction
- Keep the page closer to a clean project index than a formal presentation site.
- Favor clear hierarchy, restrained interaction, and quick scanning.
- Use images and videos only when they improve understanding quickly.

## Maintenance Check
- Before making a major addition, ask whether it helps the repository work better as a public project archive.
- If the change mainly serves formal application materials, it likely belongs in the resume or PDF instead.
